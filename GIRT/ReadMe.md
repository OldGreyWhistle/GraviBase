## Methodology summary

**Stream 3 corpus-bounded discipline:** trial work is confined to a defined document corpus. No adjacent-literature search unless authorised (modified per CD-OP-016 corpus-currency due-diligence standing rule for periodic verification on author-supplied review lists). Honest reporting of negative findings required. Significance thresholds defined (5σ / 4σ / 2σ–3σ). Variable speed of light permanently rejected as framework element. Parallel-track concepts tracked separately, not conflated with active modules.

**Modal-status preservation:** statement modal status (committed / possible / open / disclaimed) preserved across translations, integrations, and revisions. Promoting hedged claims to commitments is a logged failure mode (framing-bias correction #4).

**Audit-trail integrity:** no in-place edits. Superseded documents retain content unchanged; successors carry new version numbers. Filenames containing historical identifiers (e.g., the v1.1 master VC retaining S3M5S1) are kept verbatim.

**Framing-bias correction:** bidirectional. Both pro-framework and anti-framework pre-judging are symmetric failure modes. Four corrections logged in trial work.

**Controlled-synthesis-injection vs uncontrolled-synthesis-creep:** explicit, recorded introduction of new framework commitments is controlled-injection (documented in Introduction Letter and VC Amendments). Inadvertent introduction is creep, actively guarded against.

For full operational definitions see [Research Toolkit v1.1](./UMF_Trial_S3M0S1_Research_Toolkit_v1_1.md) §2.

**For general public / first-time visitors:**
Start with the [parent ReadMe](../ReadMe.md), then this file's "Brief description" via the Toolkit overview.

**For scientific researchers:**
Read order 1–8 above. The Synthesis Paper Scaffold v8 in `./companion-documents/` is the substantive content map; deep-review reports in `./deep-review-reports/` give per-paper provenance.

**For engineers / methodology practitioners:**
Read the Research Toolkit v1.1 (this folder). It is program-portable: methodology, protocols, navigation-layer architecture, version-control workflow, and cross-program portability are usable for other corpus-bounded research programmes.

**For LLM agents operating on this corpus:**
Read Introduction Letter v2 (mandatory bundle per Amendment 8 §K). The NOTICE in the Introduction Letter binds: LLMs operating in the UMF framework treat Toolkit + Addendum as operative; LLMs operating outside treat the Toolkit as operative and the Addendum as illustrative only.
